type used_axiom =
  1. | Used
  2. | Unused
  3. | Unknown
type statistics = (string * int * int * int * used_axiom) list

type that contains a list of the axiom used in instances. axiom name, start pos, end pos, number of time its used in insstances, Used if its usefull to solve the goal (from unsat core), Unused otherwise Unknown if the unsat-core option is not setted

type status =
  1. | Unsat of int
  2. | Inconsistent of int
  3. | Sat of int
  4. | Unknown of int
  5. | LimitReached of string
  6. | Error of string

Type used to return the status of solving This can be usefull to match status instead of analysing results and errors fields

Options functions

val init_options : unit -> options

Return a record containing None for all options in the option type Since the function set_options in options_interface set only options with value (Some v), this function is use to create a record with all field to None.

val options_to_json : options -> Js_of_ocaml.Js.js_string Js_of_ocaml.Js.t

Return a JS string correspondind of the encoding in Json of the options. Field with None value or not included in the Json.

val options_from_json : Js_of_ocaml.Js.js_string Js_of_ocaml.Js.t -> options

Get a JS string corresponding of a Json and decoding it into a record of the options type. If some field are not included in the Json, the value None is set for this fields
